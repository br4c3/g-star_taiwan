전체 논문선행연구인천 단독시흥 단독인천–시흥 연계정량자료필수 요건대회 목적
MANDATORY REQUIREMENTS

지역문제 해결 과제 필수 요구사항

모든 지역 아이디어와 최종 제안서에 공통으로 적용하는 기준입니다.

지역의 자산을 활용하고(로컬), 사회적 문제를 해결하는 미션을 세우며(소셜), 사용자 중심 경험을 설계하고(디자인씽킹), 지속 가능한 수익구조와 기술로 구현해야 합니다.

1. 명확한 소셜미션

프로젝트의 존재 이유, 대상 사용자, 해결할 문제와 만들 변화를 한 문장으로 정의합니다.

문장 구조

우리는 [지역자산]을 활용해 [대상 사용자]가 겪는 [사회문제]를 해결하고 [측정 가능한 변화]를 만든다.

2. ABCD와 커뮤니티매핑

결핍보다 개인·관계·기관·물리·자연·문화·경제 자산을 먼저 조사합니다.

  • 자산명·유형·위치·접근수단
  • 운영주체·시간·비용·주요 이용자
  • 현재 활용 수준과 지역 고유 이야기
  • 사용자 문제와의 연결 가능성
  • 출처·최종 확인일·주민 제보·운영자 확인

3. 검증 가능한 4대 가설

가설필수 질문
페르소나누가 어떤 상황에서 반복적으로 문제를 겪는가?
문제WHO–WHY–WHAT으로 원인과 행동상 손실을 설명할 수 있는가?
솔루션제안이 어떤 실제 행동을 바꾸는가?
시장규모TAM·SAM·SOM의 포함조건과 지불자가 명확한가?

4. 디자인씽킹과 고객여정

이용 전–탐색–선택·예약–현장 이용–이용 후의 행동, 감정, Pain Point와 측정지표를 작성합니다. 사용자뿐 아니라 상인과 관리자 경험도 설계합니다.

5. 소셜 비즈니스 모델

BMC 9요소에 목적과 임팩트를 추가합니다.

  • 고객 세그먼트·가치제안·채널·고객관계
  • 수익원·핵심자원·핵심활동·파트너·비용구조
  • 해결할 사회문제와 측정 가능한 결과
  • 지원금 이후의 반복매출과 운영비

6. 기술적 타당성과 UML

  • 유즈케이스: 사용자·상인·관리자·협력기관 기능
  • 클래스: 사용자·지역자산·상점·코스·추천·쿠폰·임팩트 데이터
  • 시퀀스: 입력→검색→추천→방문·이용→기록
  • 개인정보, 데이터 최신성, 라이선스와 부정사용 방지

7. 로컬크리에이터 정체성

전국 어디서나 가능한 플랫폼이 아니라 해당 지역에서 시작할 이유가 있어야 합니다. 지역자산을 새로운 경험·콘텐츠·상품·관계로 재해석하고 지역 공급자와 공동 생산합니다.
