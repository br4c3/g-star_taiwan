# 개항장 B2B 로컬기프트 과제 필수 요구사항

## 1. 소셜미션

> 개항장 소상공인의 상품과 지역 이야기를 B2B 로컬기프트로 공동기획하여 정상가격 매출과 반복거래를 만든다.

미션은 상품선정, 정산, 매출·임팩트 지표와 연결해야 한다.

## 2. ABCD 지역자산

- 개인: 상인·공예가·생산자의 기술과 기억
- 조직: 상점·시장·문화기관·행사·숙박 네트워크
- 공간: 개항장 건축·골목·시장·항만문화
- 경제: 기존 상품·공급설비·포장·인쇄·물류
- 문화: 교역·이주·음식·인쇄·생활사

자산지도에는 위치뿐 아니라 상품, 근거, 권리, 공급량, 리드타임과 검증상태를 기록한다.

## 3. 네 가지 가설

| 가설 | 내용 | 검증 |
|---|---|---|
| 페르소나 | 인천 행사·숙박·기업 구매담당자 | 최근 구매업무와 의사결정권 확인 |
| 문제 | 지역상품 소싱·통합거래가 어렵다 | 기존 대안·시간·실패사례 질문 |
| 솔루션 | 표준키트·통합견적·납품이 구매를 만든다 | 샘플·견적·유료발주 |
| 시장 | 접근 가능한 반복 구매기관이 존재한다 | 20곳 영업·구매시점·재주문 |

## 4. 사용자 경험

구매기관의 탐색–문의–견적–승인–발주–수령–재구매와 상점의 제안–공급확정–입고–정산을 함께 설계한다. 구매기관의 편의 때문에 상점에 과도한 맞춤·재고·외상 부담이 전가되지 않도록 서비스 블루프린트가 필요하다.

## 5. 소셜 비즈니스 모델

BMC 9요소 외에 다음을 포함한다.

- 수혜·공급자: 개항장 상점·공방
- 사회성과: 정상가격 매입액, 최초 B2B 거래, 반복공급, 매출분포
- 부정영향: 특정상점 편중, 로컬워싱, 표시위반, 재고·현금흐름 부담
- 수익: 판매마진·맞춤제작비와 실제 공헌이익

## 6. 기술과 UML

- 유즈케이스: 상품등록, 공급확인, 견적, 발주, 조달, 검수, 납품, 정산
- 클래스: Supplier, Product, GiftSet, Buyer, Quote, Order, Batch, Delivery, Settlement
- 시퀀스: 구매요청부터 재주문까지
- 기술원칙: 초기에는 CRM·QR·문서자동화만, AI·소비자앱 제외

## 7. 로컬크리에이터 정체성

기존 상품을 단순 재판매하지 않고 지역성 근거, 생산자 이야기, 표준 패키지와 B2B 판로를 결합한다. 개항장에서 검증한 소싱–권리–표준화–납품 방법이 반복 가능해야 한다.

## 8. 제출 체크리스트

- [ ] 개항장 자산·상점 후보 지도
- [ ] 상점 5곳 공급·콘텐츠 동의
- [ ] 구매담당자 인터뷰와 문제검증
- [ ] 표준키트 실물 샘플
- [ ] 가격·원가·공헌이익표
- [ ] 30·50·100세트 견적서
- [ ] B2B 고객여정·서비스 블루프린트
- [ ] 사회적 BMC와 임팩트 지표
- [ ] 주문·품질·정산 UML
- [ ] 12주 WBS와 승인 게이트
- [ ] 견적·유료발주 또는 기각 증거

## 9. 평가 통과기준

지역성, 문제근거, 지불의사, 반복공급, 단위경제성, 실행 가능성, 사회성과와 윤리·법적 위험을 각각 설명할 수 있어야 한다. 기능 수나 시장 총량보다 검증의 질을 우선한다.
