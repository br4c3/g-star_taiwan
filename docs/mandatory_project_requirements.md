# 지역문제 해결 과제 필수 요구사항

> 문서 역할: 인천 단독, 시흥 단독 또는 연계형 아이디어를 기획·평가할 때 공통으로 적용하는 기준 문서  
> 적용 범위: 자료조사, 문제정의, 서비스 기획, 비즈니스 모델, UML, 발표자료, 최종 제안서

## 1. 과제의 핵심 정의

> 지역의 자산을 활용하고(로컬), 사회적 문제를 해결하는 미션을 세우며(소셜), 사용자 중심의 경험을 설계하고(디자인씽킹), 이를 지속 가능한 수익구조와 기술로 구현하는 것(비즈니스)이 이번 과제의 필수 요건이다.

단순한 지역정보 조사, 앱 아이디어 또는 관광코스 제안만으로는 충분하지 않다. 지역성, 사회적 가치, 사용자 경험, 사업성과 기술적 실현 가능성이 하나의 논리로 연결되어야 한다.

```text
지역자산 발굴
  ↓
지역 맥락과 사용자 문제 정의
  ↓
소셜미션 수립
  ↓
4대 가설 설정·검증
  ↓
사용자 경험 설계
  ↓
소셜 비즈니스 모델 구축
  ↓
기술·운영 타당성 제시
  ↓
임팩트 측정과 성장 로드맵
```

---

## 2. 필수 요구사항 1: 명확한 소셜미션

### 요구 내용

- 기업이나 프로젝트가 왜 존재하는지 한 문장으로 정의한다.
- 해결하려는 사회문제, 대상 사용자와 만들고자 하는 변화를 포함한다.
- 미션을 선언적 구호로 끝내지 않고 서비스 기능, 수익구조, 파트너십과 임팩트 지표에 연결한다.

### 필수 결과물

```text
우리는 [지역/자산]을 활용해
[대상 사용자]가 겪는 [사회적 문제]를 해결하고
[측정 가능한 변화]를 만든다.
```

### 검토 질문

- 미션에서 지역 이름을 빼도 성립한다면 지역성이 약하지 않은가?
- 해결하려는 문제가 사용자 조사로 확인됐는가?
- 매출이 증가할수록 사회적 성과도 함께 증가하는 구조인가?
- 핵심 기능과 수익원이 미션 달성에 실제로 기여하는가?

## 3. 필수 요구사항 2: ABCD 기반 지역 맥락 분석

### 요구 내용

- 지역의 결핍보다 이미 존재하는 유·무형 자산을 먼저 조사한다.
- 자연·문화·역사·산업·공간뿐 아니라 주민의 기술, 관계망, 조직과 제도도 자산으로 본다.
- 자산의 위치, 운영주체, 이용자, 활용 수준과 연결 가능성을 커뮤니티매핑으로 기록한다.
- 주민과 운영자가 자산을 발견하고 검증하는 과정에 참여하게 한다.

### 자산 분류

| 자산 유형 | 조사 대상 예시 |
|---|---|
| 개인 자산 | 주민·근로자의 기술, 경험, 지역지식 |
| 관계 자산 | 동호회, 청년모임, 상인회, 주민 네트워크 |
| 기관 자산 | 지자체, 대학, 산업단지, 관광기관, 사회적기업 |
| 물리 자산 | 공원, 문화공간, 산업시설, 유휴공간, 교통시설 |
| 자연 자산 | 습지, 갯골, 해안, 섬, 생태계 |
| 문화 자산 | 역사, 노동과 산업의 기억, 축제, 지역 먹거리 |
| 경제 자산 | 소상공인, 제조기업, 로컬브랜드, 전통시장 |

### 커뮤니티매핑 필수 필드

- 자산명과 유형
- 위치와 접근수단
- 운영주체와 연락처
- 운영시간과 이용비용
- 주요 이용자
- 현재 활용 수준
- 지역 고유 이야기
- 사용자 문제와의 연결 가능성
- 정보 출처와 최종 확인일
- 주민 제보 및 운영자 확인 상태

### 필수 결과물

- 지역자산 지도
- 자산 목록 및 분류표
- 자산–사용자–문제 연결표
- 주민·운영자 참여 및 검증 과정

## 4. 필수 요구사항 3: 검증 가능한 4대 가설

### 4.1 페르소나 가설

누가 가장 반복적으로 문제를 겪는지 정의한다.

- 연령보다 실제 행동과 상황을 중심으로 작성한다.
- 근무시간, 이동수단, 거주지, 동행자, 가용시간과 현재 대안을 포함한다.
- 넓은 사용자군을 하나의 페르소나로 묶지 않는다.

### 4.2 문제 가설

사용자가 어떤 상황에서 왜 문제를 겪는지 정의한다.

```text
WHO: 문제를 겪는 구체적인 사용자
WHY: 문제를 발생시키는 원인
WHAT: 사용자가 겪는 행동상 불편과 손실
```

### 4.3 솔루션 가설

제안한 해결책이 어떤 사용자 행동을 바꿀 것인지 정의한다.

```text
[기능 또는 경험]을 제공하면
[사용자 행동]이 바뀌고
[지역·사회적 결과]가 발생할 것이다.
```

### 4.4 시장규모 가설

- 전체 주민을 시장규모로 사용하지 않는다.
- TAM, SAM, SOM의 포함조건과 중복 제거 기준을 제시한다.
- 사용자와 지불자를 분리한다.
- 인구, 취업자, 사업체와 방문자 통계의 기준연도를 맞춘다.

### 필수 결과물

| 가설 | 현재 근거 | 검증 방법 | 성공·기각 기준 | 결과 |

## 5. 필수 요구사항 4: 디자인씽킹 기반 사용자 경험

- 사용자 인터뷰와 관찰을 통해 실제 행동을 조사한다.

## 7. 필수 요구사항 6: 기술적 타당성과 UML

### 요구 내용

- 기술을 목적이 아니라 소셜미션을 구현하는 수단으로 사용한다.
- 사용자, 지역상인, 관리자와 협력기관의 기능을 유즈케이스로 표현한다.
- 핵심 데이터 구조를 클래스 다이어그램으로 표현한다.
- 주요 서비스 흐름을 시퀀스 다이어그램으로 표현한다.

### 필수 UML

1. **유즈케이스 다이어그램**
   - 사용자별 기능과 시스템 경계
2. **클래스 다이어그램**
   - User, LocalAsset, Store, Route, Recommendation, Coupon, Review, ImpactData 등
3. **시퀀스 다이어그램**
   - 사용자 입력 → 자산 검색 → 추천 생성 → 방문·이용 → 데이터 기록

### 함께 검토할 기술 조건

- 공공데이터의 최신성, 결측치와 이용허락
- 위치정보와 개인정보 최소 수집
- 추천 결과의 설명 가능성
- 잘못된 영업시간과 폐업 정보 처리
- QR·쿠폰 부정사용 방지
- 상인·주민 제보의 승인과 수정 이력
- 모바일 웹 기반 MVP 가능성

## 8. 필수 요구사항 7: 로컬크리에이터 정체성

### 요구 내용

- 지역의 자연·문화·산업적 특성을 사업의 핵심 소재로 사용한다.
- 전국 어디서나 가능한 플랫폼이 아니라 해당 지역에서 시작해야 하는 이유를 제시한다.
- 기존 자산을 새로운 경험, 콘텐츠, 상품 또는 관계로 재해석한다.
- 지역 공급자와 공동으로 가치를 생산한다.

### 가능한 로컬 비즈니스 유형

- 무형상품: 지역 코스, 해설, 큐레이션
- 공간: 지역상점, 문화공간, 산업·생태 거점
- 미디어: 지역자산 지도, 인터뷰, 지역 이야기
- 체험: 생태, 제조, 역사, 먹거리, 해양레저
- 유형상품: 제조기술과 지역문화를 결합한 로컬상품

### 성장 로드맵

1. 지역자산 조사 및 문제 검증
2. 좁은 생활권에서 MVP 실험
3. 지역상인·기관 파트너십 구축
4. 반복수익과 임팩트 검증
5. 인접 권역 또는 유사 지역으로 확장
6. 로컬크리에이터·사회적기업·관광벤처 등 지원사업 연계

지원사업은 초기 실증과 성장수단으로 활용하되 지속 가능한 수익원으로 간주하지 않는다.

## 10. 아이디어 평가표

각 지역 아이디어를 1~5점으로 평가한다.

| 평가항목 | 질문 | 가중치 |
|---|---|---:|
| 지역성 | 해당 지역 고유 자산이 핵심 해결수단인가? | 15% |
| 사회문제 | 구체적 사용자의 반복 문제가 확인되는가? | 15% |
| 소셜미션 | 목적과 변화가 한 문장으로 명확한가? | 10% |
| 자산 활용 | 자산과 문제의 연결 논리가 강한가? | 10% |
| 사용자 경험 | 고객여정과 Pain Point가 구체적인가? | 10% |
| 검증 가능성 | 대회 기간 안에 행동실험이 가능한가? | 10% |
| 사업성 | 지불자와 반복수익이 명확한가? | 10% |
| 임팩트 | 증분 사회성과를 측정할 수 있는가? | 10% |
| 기술 타당성 | 데이터·운영·UML로 구현을 설명할 수 있는가? | 5% |
| 성장성 | 지역 브랜드로 성장할 경로가 있는가? | 5% |

### 통과 기준

- 총점이 높아도 지역성, 사회문제, 소셜미션 중 하나가 2점 이하라면 보완한다.
- 공개 통계만 있고 사용자 검증이 없으면 문제 가설은 확정하지 않는다.
- 기능은 많지만 지불자나 임팩트가 불명확하면 MVP 범위를 축소한다.
