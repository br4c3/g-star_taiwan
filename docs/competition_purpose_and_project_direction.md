# 인천·시흥 지역문제 해결 대회 목적 및 과제 방향

> 작성 기준: 현재까지 전달된 과제 설명  
> 참고: 공식 대회 공고문과 평가표가 확보되면 명칭, 배점, 제출 형식은 추가로 조정해야 한다.

## 1. 대회의 목적

이번 대회의 목적은 인천이나 시흥의 문제를 조사한 뒤 단순한 해결 아이디어를 제안하는 데 있지 않다.

지역이 이미 보유한 자연, 문화, 역사, 산업, 공간, 사람, 조직과 같은 자산을 발견하고, 그 자산을 실제 지역문제 해결에 활용해 사용자에게 필요한 서비스와 지속 가능한 사업으로 구체화하는 것이 핵심이다.

### 한 문장 정의

> 인천 또는 시흥의 지역자산을 발굴하고, 이를 구체적인 사용자의 지역문제 해결과 연결하여 사회적 가치와 사업적 지속 가능성을 함께 갖춘 서비스로 만드는 것이 이번 대회의 목적이다.

### 이 대회가 요구하는 결과

좋은 아이디어 하나를 제시하는 것이 아니라 다음 내용이 하나의 논리로 연결되어야 한다.

```text
지역자산
  ↓
구체적인 사용자 문제
  ↓
검증 가능한 해결 가설
  ↓
사용자 중심 서비스
  ↓
수익과 운영 구조
  ↓
사회적 성과
  ↓
기술·운영 가능성
```

## 2. 대회에서 답해야 할 핵심 질문

### 2.1 지역성

- 왜 인천 또는 시흥이어야 하는가?
- 이 지역이 가진 고유한 자산은 무엇인가?
- 다른 지역에서 쉽게 복제하기 어려운 요소는 무엇인가?

### 2.2 문제성

- 실제로 문제를 겪는 사람은 누구인가?
- 그 문제는 얼마나 자주 발생하는가?
- 현재 사용자는 어떤 방법으로 문제를 해결하고 있는가?
- 기존 해결 방법은 왜 충분하지 않은가?

### 2.3 자산 활용성

- 발굴한 지역자산이 문제 해결에 어떤 역할을 하는가?
- 현재 충분히 활용되지 못하는 이유는 무엇인가?
- 자산과 사용자 사이의 단절을 어떻게 해결할 것인가?

### 2.4 사용자성

- 사용자는 서비스를 언제, 어디에서, 왜 이용하는가?
- 서비스를 이용하기 전과 후의 행동은 어떻게 달라지는가?
- 사용자가 실제로 방문하거나 구매하거나 참여하는가?

### 2.5 사업성

- 서비스를 사용하는 사람과 비용을 지불하는 사람은 누구인가?
- 지역상인, 기업, 기관은 왜 비용을 지불하는가?
- 반복적으로 발생할 수 있는 수익은 무엇인가?
- 지원사업이 끝난 뒤에도 운영할 수 있는가?

### 2.6 사회성

- 서비스가 지역에 어떤 변화를 만드는가?
- 지역 내 소비, 지역자산 이용, 주민 참여 등이 실제로 증가하는가?
- 그 변화를 어떤 지표로 측정할 것인가?

### 2.7 실현 가능성

- 필요한 지역정보와 데이터를 확보할 수 있는가?
- 현재 기술로 MVP를 만들 수 있는가?
- 누가 데이터를 관리하고 제휴처를 운영하는가?
- 개인정보, 위치정보, 공공데이터 이용조건을 지킬 수 있는가?

## 3. 인천–시흥 연계에 대한 원칙

인천과 시흥을 반드시 하나의 사업으로 연결할 필요는 없다.

두 지역이 인접하고 일부 생활권이 연결되어 있다는 점은 활용 가능한 기회지만, 대회의 필수조건은 아니다. 조사 결과 한 지역에 집중하는 것이 문제와 자산을 더 선명하게 보여준다면 인천 단독 또는 시흥 단독으로 진행해도 된다.

### 기본 원칙

> 인천과 시흥을 먼저 연결하기로 결정한 뒤 이유를 찾지 않는다. 사용자 행동과 지역자산을 조사한 결과 두 지역의 연결이 문제 해결에 필요할 때만 연계한다.

### 선택 가능한 범위

| 사업 범위 | 선택하기 적합한 상황 | 장점 | 주의점 |
|---|---|---|---|
| 인천 단독 | 인천의 특정 자산과 사용자가 명확한 경우 | 조사와 제휴 범위를 집중할 수 있음 | 인천 전체를 대상으로 하면 다시 범위가 넓어짐 |
| 시흥 단독 | 시흥의 생태·산업·청년 문제가 더 선명한 경우 | 지역 정체성과 현장실험을 강조하기 좋음 | 사용자와 지불자 규모를 검증해야 함 |
| 인천–시흥 연계 | 사용자의 실제 이동과 자산 이용이 경계를 넘는 경우 | 행정경계를 넘는 차별성을 만들 수 있음 | 데이터·파트너·이동 동선이 복잡해짐 |

### 연계형을 선택할 조건

다음 내용이 인터뷰나 데이터로 확인될 때 연계형을 선택한다.

- 핵심 사용자가 두 지역 사이를 일상적으로 이동한다.
- 한 지역의 자산만으로는 사용자 문제를 충분히 해결하기 어렵다.
- 양쪽 자산을 연결했을 때 새로운 경험이나 소비가 발생한다.
- 이동시간과 교통수단이 실제 이용에 무리가 없다.
- 양쪽 지역의 상인과 기관이 공동 실험에 참여할 수 있다.
- 행정구역별 정보 분리가 실제 사용자 불편으로 확인된다.

이 조건이 확인되지 않으면 한 지역의 한 생활권에 집중하는 것이 더 타당하다.

## 4. 지역 선택 기준

인천 단독, 시흥 단독, 연계형 가운데 어느 방향이 적합한지는 다음 기준으로 평가한다.

| 판단 기준 | 확인할 질문 | 가중치 예시 |
|---|---|---:|
| 문제 선명도 | 반복적인 문제를 겪는 사용자가 명확한가? | 25% |
| 자산 고유성 | 이 지역만의 자산과 이야기가 있는가? | 20% |
| 사용자 접근성 | 인터뷰와 테스트 참여자를 구할 수 있는가? | 15% |
| 실험 가능성 | 대회 기간 안에 행동 변화를 측정할 수 있는가? | 15% |
| 파트너 가능성 | 상인·기관·운영자와 협업할 수 있는가? | 10% |
| 데이터 확보성 | 통계와 지역정보를 확보할 수 있는가? | 10% |
| 확장 가능성 | 초기 성공 후 인접 지역으로 확장할 수 있는가? | 5% |

점수보다 중요한 것은 선택한 지역과 범위의 이유를 사용자 문제와 지역자산으로 설명하는 것이다.

## 5. 권장 과제 진행 과정

### 1단계. 지역 후보 선정

- 인천과 시흥의 후보 생활권을 2~3곳 선정한다.
- 지역별 대표자산, 사용자, 접근성을 비교한다.
- 처음부터 인천·시흥 전역을 대상으로 삼지 않는다.

### 2단계. 지역자산 조사

- 자연, 문화, 산업, 상권, 공간, 사람과 조직을 조사한다.
- 위치, 운영시간, 운영주체, 이용자, 활용 수준을 지도에 기록한다.
- 주민과 상인의 제보를 받아 현장 정보와 차이를 확인한다.

### 3단계. 사용자 문제 발견

- 지역 근로자, 주민, 대학생, 방문객을 인터뷰한다.
- 의견보다 최근 실제 행동을 조사한다.
- 사용자가 문제를 겪는 시간, 장소, 빈도와 현재 대안을 확인한다.

### 4단계. 핵심 가설 설정

- 페르소나 가설: 가장 반복적으로 문제를 겪는 사람은 누구인가?
- 문제 가설: 사용자가 해결하지 못한 핵심 문제는 무엇인가?
- 해결 가설: 어떤 서비스가 사용자의 행동을 바꿀 것인가?
- 시장 가설: 사용자는 얼마나 존재하고 누가 비용을 지불할 것인가?

### 5단계. 작은 실험

- 완성된 앱보다 모바일 웹, 지도, 카카오톡 링크 등으로 시작한다.
- 지역자산 30~50개와 1시간·2시간 코스를 구성한다.
- 추천 확인, 길찾기, 방문, 쿠폰 사용과 재사용을 측정한다.

### 6단계. 사업모델 설계

- 사용자, 수혜자와 지불자를 구분한다.
- 상인 수수료, 예약, 기업복지, 지역상품 등 수익원을 검증한다.
- 개발비뿐 아니라 현장조사, 데이터 검수, 제휴 관리비를 포함한다.

### 7단계. 사회성과와 기술 타당성 정리

- 지역 내 증분 소비, 신규 방문, 재방문과 자산 인지도를 측정한다.
- 서비스 구조, 데이터 흐름, 주요 사용자 기능을 UML로 표현한다.
- 개인정보, 정보 최신성, 운영 책임과 생태적 영향을 함께 검토한다.

## 6. 예상 평가 관점

공식 평가표가 없는 현재 단계에서는 다음 평가 논리를 기준으로 준비한다.

| 평가 관점 | 좋은 제안의 기준 | 대표 결과물 |
|---|---|---|
| 지역성 | 지역 고유 자산을 구체적으로 활용함 | 지역자산 지도 |
| 문제성 | 명확한 사용자의 반복 문제를 제시함 | 인터뷰와 문제정의 |
| 혁신성 | 기존 플랫폼과 다른 해결 방식을 제시함 | 경쟁·대안 분석 |
| 사용자성 | 사용 전후의 행동 변화를 설명함 | 페르소나와 고객여정 |
| 검증성 | 주장을 인터뷰와 행동으로 확인함 | 가설 검증표 |
| 사업성 | 지불자와 반복수익이 명확함 | 소셜 BMC와 수익구조 |
| 사회성 | 측정 가능한 지역 변화를 만듦 | 임팩트 지표 |
| 실현 가능성 | 기술과 운영 구조가 현실적임 | MVP, UML, 데이터 구조 |
| 성장성 | 검증 후 확장 조건이 명확함 | 단계별 로드맵 |

## 7. 과제에서 피해야 할 주장

| 과장된 주장 | 보다 정확한 표현 |
|---|---|
| 청년들이 지역을 떠나고 있다 | 청년의 지역 내 여가·소비 행동을 조사한다 |
| 지역정보가 부족하다 | 정보가 사용자 상황에 맞게 연결되는지 검증한다 |
| 인천과 시흥은 하나의 생활권이다 | 일부 연계 계획이 있으며 실제 이동을 추가 확인한다 |
| 앱을 만들면 지역경제가 활성화된다 | 추천이 신규 방문과 증분 소비를 만드는지 실험한다 |
| 방문객이 많으므로 시장이 크다 | 목표 사용자와 실제 지불자를 기준으로 시장을 계산한다 |
| 체류시간이 늘면 청년이 정착한다 | 단기 체류와 장기 정착 효과를 구분한다 |
| 주민 참여로 정보 문제가 해결된다 | 제보 참여율, 정확도와 검수비용을 측정한다 |
| 지원사업에 선정되면 지속 가능하다 | 지원금 이후의 반복매출과 운영비를 검증한다 |

## 8. 현재 단계의 권장 방향

현 단계에서는 인천–시흥 연계형 플랫폼을 먼저 확정하지 않는다.

다음 순서가 적절하다.

1. 인천과 시흥에서 각각 한 개의 후보 생활권을 선정한다.
2. 지역자산과 인터뷰 접근성을 비교한다.
3. 가장 강한 사용자 문제를 가진 지역 하나를 초기 대상지로 선택한다.
4. 한 지역 안에서 문제와 해결 가설을 먼저 검증한다.
5. 사용자 이동과 자산 활용에 필요성이 확인되면 인접 지역으로 확장한다.

### 초기 후보 예시

- 인천: 남동산단–소래 생활권
- 시흥: 시흥스마트허브–오이도 생활권
- 연계 검토: 소래–월곶 또는 송도–배곧

후보지는 현재 자료를 바탕으로 한 예시이며, 현장조사 전 확정 대상은 아니다.

## 9. 최종 과제 방향 문장

> 인천 또는 시흥의 특정 생활권이 가진 산업·생태·문화·상권 자산을 주민 참여형 조사로 발굴하고, 그 자산을 충분히 활용하지 못하는 구체적인 사용자의 문제를 찾는다. 이후 사용자 인터뷰와 작은 현장실험을 통해 해결 가설을 검증하고, 실제 방문·소비·참여 변화를 만드는 서비스와 지속 가능한 소셜 비즈니스 모델을 설계한다. 인천–시흥 연계는 사용자의 이동과 문제 해결에 필요하다는 근거가 확인될 때 확장 방향으로 선택한다.

### 핵심 원칙

> 지역을 연결하는 것보다 지역자산과 사용자 문제를 정확히 연결하는 것이 먼저다.
